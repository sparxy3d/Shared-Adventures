Update the existing Free Spirit MVP to pivot from a traditional activity booking marketplace into a “What should we do now?” discovery-first platform.

Important:
- DO NOT rebuild the whole app from scratch
- REUSE the existing project structure, booking foundation, auth, database, and current activity pages where possible
- This is a product pivot and homepage / flow redesign
- Keep the existing booking capability, but make discovery the primary experience

New product direction:
Free Spirit is a platform that helps people quickly decide what to do right now, tonight, or this weekend based on location, time, and group size.

Core value proposition:
- Help groups decide what to do
- Suggest activities that are open now or starting soon
- Make spontaneous planning easier
- Booking is secondary to discovery

PRIMARY UX CHANGE:
The homepage should no longer feel like a generic marketplace first.
It should feel like a decision engine.

Main homepage hero:
Headline:
What should we do right now?

Subheadline:
Find activities near you that are open, available, and worth doing together.

Inputs in hero:
1. Location
   - text input with placeholder: Enter city or use current location
   - include “Use my location” button if easy
2. Group size
   - chips or segmented options:
     1
     2
     3–5
     6+
3. Time
   - segmented options:
     Now
     Tonight
     Weekend

Primary CTA:
Show Ideas

Secondary CTA:
Surprise Us

Top-level product goal:
Transform the current platform into a discovery-first activity finder with lightweight booking.

4) ADD “SURPRISE US” FLOW
Implement a lightweight surprise mode.

Behavior:
- Use the selected location + time mode + group size
- Show one random matching activity suggestion

- Display a fun loading state:
  Finding something fun...
- Then show a result card with:
  title
  image
  location
  open now / next session
  button: View Details

This can be implemented with simple random selection logic from filtered activities.

5) DATA MODEL ENHANCEMENTS
Do not destroy the current schema.

If not already present, add fields to activities / experiences:
- country
- city or region
- ideal_for_tags (friends, couples, families, teams, solo)
- duration_minutes
- opening_hours_text OR structured open hours
- is_open_now logic support
- next_session_text or session support

If easier, start with simple fields and computed labels instead of full scheduling complexity.

6) SIMPLE “OPEN NOW” / “STARTING SOON” LOGIC
Implement a practical MVP approach.

If structured opening hours exist:

If not:
- use simple fallback fields like:
  open_time
  close_time
  next_session_text

For MVP, it is acceptable to support:
- open now
- opens later
- next session at X

Do NOT overengineer real-time scheduling.
